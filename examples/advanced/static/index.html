Static fallback

You're looking at a file served by api/fallback.rs, which attaches a ServeDir over this folder's subtree. Any request that doesn't match a route.rs falls back to a file in this directory — including / itself, which has no handler and so resolves to this index.html.

Try the routes too: GET /ping is handled by a route.rs, while /users is guarded by api/users/intercept.rs (send Authorization: Bearer password).
